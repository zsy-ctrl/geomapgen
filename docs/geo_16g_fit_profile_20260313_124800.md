## Clarification

The model does **not** take both the raw cropped patch and the resized patch as two separate inputs.

Current training flow:

- read cropped patch from GeoTIFF
- resize/pad it to `image_size`
- send only the resized tensor into the model

The raw cropped patch is only kept for cache or artifact/debug export.

## Update

To further reduce memory pressure, the LoRA default model input size was reduced again:

- `data.image_size = 128`
