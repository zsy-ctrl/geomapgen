# Geo Smoke Debug 20260312 191200

## Symptom

- Smoke predict wrote empty `Lane.geojson`.
- `Intersection.geojson` was missing in the earlier failing path.

## Root Cause

1. `predict_geo_vector.py` used to raise immediately after the first failed task, so a lane failure prevented `Intersection.geojson` from being written.
2. The structured decode sequence originally placed `properties` before point coordinates:
   - `<obj> <geom> <src> <cut_in> <cut_out> <prop> ... <prop_end> <pt> ...`
3. Tiny smoke checkpoints were spending the whole decode budget inside the free-form property span and never reached any point tokens.
4. Result:
   - parser saw an object header
   - parser saw no valid points
   - final `.geojson` had empty `features`

## Fixes

1. `predict_geo_vector.py`
   - write all task outputs first
   - collect failed tasks
   - raise only after both `Lane.geojson` and `Intersection.geojson` have been written
2. `GeoMapGrammarHelper`
   - constrain generation to the structured map grammar
   - cap property-span length with `decode.max_prop_tokens`
3. `GeoCoordTokenizer`
   - reorder training/inference map sequence to geometry-first:
     - `<obj> <geom> <src> <cut_in> <cut_out> <pt> ... <prop> ... <prop_end> <obj_end>`
   - decoder now tolerates both old and new order when parsing object bodies
