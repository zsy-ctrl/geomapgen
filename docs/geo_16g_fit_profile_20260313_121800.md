# Geo 16G Fit Profile 20260313 121800

## Goal

Tighten the default LoRA training profile so it is more likely to run on a 16 GB GPU.

## Updated 16G-Oriented LoRA Defaults

- `data.image_size = 308`
- `serialization.coord_bins = 768`
- `serialization.tasks.lane.max_features = 48`
- `serialization.tasks.intersection.max_features = 24`
- `text.prompt_max_tokens = 96`
- `text.state_max_tokens = 256`
- `text.target_max_tokens = 768`
- `state_update.max_features = 8`
- `state_update.anchor_max_points = 4`
- `model.sat_token_hw = [2, 2]`
- `model.lora.r = 8`
- `model.lora.alpha = 16`
- `model.lora.target_modules = [q_proj, k_proj, v_proj, o_proj]`
- `model.gradient_checkpointing = true`
- `train.amp = true`

## Launcher Updates

LoRA launch scripts now default to the same tighter profile and also export:

- `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True`

Files updated:

- `scripts/run_geo_lora_train.sh`
- `scripts/run_geo_lora_train.ps1`

## Tradeoff

This profile reduces memory pressure at the cost of some capacity:

- lower image detail
- fewer visual tokens
- fewer task objects per patch
- shorter map/state sequences
- lower LoRA rank

If training is stable, these values can be relaxed gradually.
